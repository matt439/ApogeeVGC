# Equivalence Test Bugfixes Index

Documented bugfixes discovered and resolved through equivalence testing of the C# battle simulator against Pokemon Showdown.

- [ability-item-state-reinit-switchin](ability-item-state-reinit-switchin.md) -- AbilityState and ItemState were not re-initialized on switch-in, causing per-appearance flags (e.g., Protean) to persist across switches.
- [absorb-abilities-spurious-priority](absorb-abilities-spurious-priority.md) -- Water Absorb and Volt Absorb had spurious `priority=1` on OnTryHit, breaking field-position ordering against Storm Drain and Motor Drive on spread moves.
- [addvolatile-nullrelayvar-mapping](addvolatile-nullrelayvar-mapping.md) -- AddVolatile's NullRelayVar return was mapped to undefined instead of null, causing didSomething to be promoted to true when Aroma Veil blocked a volatile.
- [attrlastmove-still-empty-target](attrlastmove-still-empty-target.md) -- The `[still]` tag on move protocol lines was missing an empty target field when the parts array had only 4 segments.
- [auroraveil-double-application](auroraveil-double-application.md) -- Aurora Veil's `OnAnyModifyDamage` handler fired twice when both sides had the condition, applying damage reduction twice.
- [berserk-blocks-berry-on-confusion](berserk-blocks-berry-on-confusion.md) -- Berserk ability did not block berry consumption on confusion self-hit, causing PRNG divergence from Showdown.
- [chainmodify-fixedpoint-corruption](chainmodify-fixedpoint-corruption.md) -- ChainModify returned its fixed-point integer result, corrupting relay variables and causing exponential damage inflation (e.g., Glaive Rush 80 -> 16384).
- [checkfainted-missing-status-clear](checkfainted-missing-status-clear.md) -- CheckFainted did not clear the status of fainted Pokemon, causing stale status to persist and affect subsequent event handlers.
- [choosemove-locked-move-target-validation](choosemove-locked-move-target-validation.md) -- Locked moves (e.g., Recharge) were rejected by ChooseMove because target validation ran before the locked-move bypass.
- [choosemove-struggle-rejection](choosemove-struggle-rejection.md) -- Explicit `move struggle` choices were rejected by the move parser because Struggle is not listed in request available moves.
- [clear-weather-terrain-effectstate](clear-weather-terrain-effectstate.md) -- ClearWeather/ClearTerrain did not write back cleared EffectState to the Field object, leaving stale Duration values that caused phantom residual handlers and spurious PRNG calls.
- [collision-course-electro-drift-effectiveness](collision-course-electro-drift-effectiveness.md) -- Collision Course and Electro Drift checked raw effectiveness enum values instead of calling `.ToModifier()` for the super-effective boost.
- [combineresults-js-type-priority](combineresults-js-type-priority.md) -- CombineResults was missing JS type priority logic, and moveSucceeded treated damage of 0 as failure, preventing U-turn through Disguise from triggering selfSwitch.
- [combineresults-null-spread-tag](combineresults-null-spread-tag.md) -- CombineResults had no representation for JS null, causing blocked spread move targets to incorrectly retain their `[spread]` damage tag.
- [commit-choices-extra-actionspeed](commit-choices-extra-actionspeed.md) -- CommitChoices called GetActionSpeed in a redundant loop, producing 4 spurious PRNG calls that desynchronized speed-tie resolution.
- [confusion-magic-guard-effecttype](confusion-magic-guard-effecttype.md) -- Confusion self-hit damage was blocked by Magic Guard because it used EffectType.Condition instead of EffectType.Move for the pseudo-move.
- [constant-union-event-handlers](constant-union-event-handlers.md) -- Constant-value event handlers in union types (e.g., `OnCriticalHit = false`) were silently ignored due to a missing interface implementation.
- [curse-move-target-type](curse-move-target-type.md) -- Curse showed the wrong target type in request data for non-Ghost users, causing choice validation to incorrectly require a target location.
- [dancer-effectorder-sort-direction](dancer-effectorder-sort-direction.md) -- Dancer ability sorted tied-speed Pokemon by `effectOrder` ascending instead of descending, causing the wrong Dancer to activate first.
- [dictionary-iteration-order](dictionary-iteration-order.md) -- C# `Dictionary` iteration order diverged from JavaScript object insertion order after deletions, causing SpeedSort shuffles to differ for tied handlers.
- [didsomething-null-to-false](didsomething-null-to-false.md) -- Null didSomething was passed directly to CombineResults instead of being converted to false, causing blocked targets to not be filtered from the `[spread]` tag list.
- [disablemove-duplicate-runevent](disablemove-duplicate-runevent.md) -- `GetMoveRequestData()` redundantly re-ran `RunEvent(DisableMove)`, consuming extra PRNG calls via SpeedSort shuffles and desynchronizing damage randomizer values.
- [effectorder-counter-divergence](effectorder-counter-divergence.md) -- The effectOrder counter diverged (0,2,4,6... vs 0,1,2,3...) due to double-increment on switch-in, and protect-like moves created volatiles in the wrong order.
- [effectstate-shallowclone-incomplete](effectstate-shallowclone-incomplete.md) -- `EffectState.ShallowClone()` only copied ~8 fields, causing Shed Tail/Baton Pass transferred Substitute volatiles to have 0 HP.
- [electroshot-spurious-duration](electroshot-spurious-duration.md) -- ElectroShot's volatile had a spurious `Duration = 2` causing an extra residual tick and PRNG desynchronization.
- [encore-onstart-null-return](encore-onstart-null-return.md) -- Encore's `OnStart` handler returned null (interpreted as failure), causing `AddVolatile` to immediately remove the Encore volatile.
- [fainted-handler-skip-runevent](fainted-handler-skip-runevent.md) -- Fainted Pokemon's event handlers were not skipped in RunEvent, and Well-Baked Body had incorrect `priority=1` on OnTryHit.
- [fieldevent-ability-suppression](fieldevent-ability-suppression.md) -- FieldEvent's residual loop did not check ability/item suppression, causing abilities like Hunger Switch to fire on transformed Ditto.
- [flash-fire-spread-accuracy](flash-fire-spread-accuracy.md) -- Flash Fire did not set `move.accuracy=true` after absorbing a Fire move, causing extra PRNG accuracy rolls for remaining spread move targets.
- [forceswitch-int-vs-bool](forceswitch-int-vs-bool.md) -- ForceSwitch used `CanSwitch`'s raw int return instead of converting to bool, preventing the `[still]` tag on failed force-switches.
- [fractional-speed-integer-division](fractional-speed-integer-division.md) -- `IPriorityComparison.Speed` was `int`, truncating fractional speed adjustments to zero and breaking switch-in ability ordering for same-speed Pokemon.
- [from-source-effect-type-check](from-source-effect-type-check.md) -- The `[from]` attribution tag used reference equality instead of EffectType checks, causing incorrect presence or absence of attribution.
- [frozen-cure-tostring-dump](frozen-cure-tostring-dump.md) -- Defrost cure messages included the ActiveMove's `ToString()` dump instead of the move's display name.
- [full-equivalence-omnibus](full-equivalence-omnibus.md) -- Broad omnibus fix addressing action ordering, protocol formatting, event return semantics, damage attribution, and choice replay logic.
- [ghost-type-trapped](ghost-type-trapped.md) -- Ghost-type Pokemon were incorrectly trapped because the Trapped condition was missing its `ImmuneTypes` declaration.
- [grass-type-powder-immunity](grass-type-powder-immunity.md) -- Grass-type Pokemon were affected by powder effects (e.g., Effect Spore) because the Powder condition was missing `ImmuneTypes = [PokemonType.Grass]`.
- [heal-fromint-zero-falsy](heal-fromint-zero-falsy.md) -- `Heal()` returned `FromInt(0)` instead of `FromFalse()` when healing was blocked or zero, preventing callers from detecting failure via `IsFalse`.
- [hitstep-break-strict-false](hitstep-break-strict-false.md) -- hitStepMoveHitLoop used loose falsiness instead of strict `=== false` for the break check, causing premature loop exit and missing eachEvent(Update) PRNG calls.
- [hustle-chainmodify-vs-modify](hustle-chainmodify-vs-modify.md) -- Hustle used `ChainModify` instead of `Modify` for its attack multiplier, producing different intermediate rounding from Showdown.
- [levitate-moldbreaker-suppression](levitate-moldbreaker-suppression.md) -- Levitate did not check Mold Breaker suppression in IsGrounded, causing Ground moves to miss when they should have hit through the suppressed ability.
- [lockedmove-targeting-volatile](lockedmove-targeting-volatile.md) -- Two-turn moves read locked-move target from the generic TwoTurnMove volatile instead of the move-specific volatile, and Metronome's branch triggered incorrectly.
- [magic-bounce-source-effect](magic-bounce-source-effect.md) -- Magic Bounce did not correctly attribute `[from] ability: Magic Bounce` on reflected moves because it passed `battle.Effect` instead of the explicit ability reference.
- [move-log-notarget-null](move-log-notarget-null.md) -- No-target move log lines showed the user's name instead of the string `"null"` when the target was absent.
- [moveflags-shallowclone-mutation](moveflags-shallowclone-mutation.md) -- MoveFlags was shared by reference between ActiveMove and the library template, causing runtime flag mutations (e.g., Shell Side Arm contact) to leak across battles.
- [multiaccuracy-float-modifier-skip](multiaccuracy-float-modifier-skip.md) -- Multi-accuracy float accuracy incorrectly applied the RunEvent modifier (e.g., Wide Lens) by using `Math.Ceiling` before the accuracy check, when Showdown skips `FinalModify` for non-integer relay vars.
- [multiaccuracy-truncation](multiaccuracy-truncation.md) -- Multi-accuracy truncation via `(int)` cast lost fractional accuracy for Triple Axel and Population Bomb; fixed with `Math.Ceiling`.
- [mustrecharge-beforemove-null](mustrecharge-beforemove-null.md) -- MustRecharge's OnBeforeMove returned C# `null` instead of `NullRelayVar`, failing to block move execution on the recharge turn.
- [natural-cure-status-protocol-id](natural-cure-status-protocol-id.md) -- Natural Cure logged C# enum names (e.g., `"Burn"`) instead of Showdown protocol status abbreviations (e.g., `"brn"`).
- [notfail-empty-onhit-mapping](notfail-empty-onhit-mapping.md) -- OnHit's NOT_FAIL (empty string) return was mapped to null, causing SingleEvent to return true and preventing hitStepMoveHitLoop from breaking correctly.
- [ogerpon-mask-base-species-chain](ogerpon-mask-base-species-chain.md) -- Ogerpon mask items could be removed because the OnTakeItem check did not walk the species chain to the root `Ogerpon` base species.
- [on-modify-move-param-swap](on-modify-move-param-swap.md) -- The equivalence batch runner used synchronous stdout/stderr reads causing deadlocks, plus leftover debug prompts blocked automated runs.
- [onally-null-relayvar](onally-null-relayvar.md) -- OnAlly* event wrappers did not convert null handler returns to NullRelayVar, allowing Flower Veil and Sweet Veil to fail to block statuses.
- [onhit-empty-hitstep-break](onhit-empty-hitstep-break.md) -- OnHit's NOT_FAIL return was mapped to NullRelayVar then converted to false, causing premature hitStepMoveHitLoop break and missing eachEvent(Update) PRNG calls.
- [ontryaddvolatile-yawn-terrain](ontryaddvolatile-yawn-terrain.md) -- Terrain OnTryAddVolatile handlers mapped null to false instead of NullRelayVar, causing Yawn to emit spurious `-fail` and `[still]` messages on terrain.
- [overcoat-missing-powder-hail-immunity](overcoat-missing-powder-hail-immunity.md) -- Overcoat was missing Powder and Hail/Snow immunity in its `OnImmunity` handler.
- [overcoat-sandstorm-immunity](overcoat-sandstorm-immunity.md) -- Overcoat was missing its OnImmunity handler for Sandstorm, causing Pokemon with Overcoat to take sand damage.
- [paradox-pokemon-missing-tags](paradox-pokemon-missing-tags.md) -- Twelve Gen 9 Paradox Pokemon (species 984-995) were missing `Tags = [SpeciesTag.Paradox]`, allowing Booster Energy to be removed by Trick.
- [preparehit-shortcircuit](preparehit-shortcircuit.md) -- Try/PrepareHit events were evaluated unconditionally instead of short-circuiting, causing Protean to activate on failed moves like Sucker Punch.
- [pressure-pp-deduction-skipped](pressure-pp-deduction-skipped.md) -- Pressure ability PP deduction was skipped because the format `sourceEffect` check prevented the handler from firing.
- [protean-libero-effectstate-vs-abilitystate](protean-libero-effectstate-vs-abilitystate.md) -- Protean/Libero stored their activation flag on event-scoped `EffectState` instead of persistent `AbilityState`, allowing multiple activations per switch-in.
- [pseudoweather-spurious-updatespeed](pseudoweather-spurious-updatespeed.md) -- Pseudo-weather add/remove spuriously called `UpdateSpeed()`, producing extra PRNG calls not present in Showdown.
- [psychicnoise-missing-handlers](psychicnoise-missing-handlers.md) -- PsychicNoise (Heal Block) was missing OnDisableMove and OnModifyMove handlers, causing PRNG divergence from missing speed-tie shuffles.
- [rage-fist-double-counting](rage-fist-double-counting.md) -- Rage Fist's `TimesAttacked` counter was incremented in both `GotAttacked()` and `HitSteps`, double-counting hits.
- [recharge-target-healblock-residual](recharge-target-healblock-residual.md) -- Recharge pseudo-move had `MoveTarget.Self` instead of `MoveTarget.None`, skipping a PRNG call for random foe targeting, and HealBlock was missing its residual order.
- [runevent-fainted-handler-skip](runevent-fainted-handler-skip.md) -- RunEvent incorrectly skipped fainted Pokemon handlers, causing issues like Magician not stealing items from fainted targets.
- [secondary-prng-substitute](secondary-prng-substitute.md) -- Secondary effect PRNG rolls were skipped entirely when Substitute absorbed a hit, desynchronizing the RNG from Showdown.
- [secondary-self-effects-substitute-followup](secondary-self-effects-substitute-followup.md) -- Self-targeting secondary effects (e.g., Aqua Step speed boost) were not applied to the attacker when the hit was absorbed by Substitute.
- [self-effects-through-substitute](self-effects-through-substitute.md) -- Self-targeting secondary effects (e.g., Close Combat stat drops) were skipped when the move hit a Substitute.
- [selfswitch-null-substitute-target](selfswitch-null-substitute-target.md) -- RunMoveEffects skipped selfSwitch processing for null (Substitute) targets, preventing Flip Turn/U-turn from switching out after hitting a Substitute.
- [setstatus-duplicate-sourceeffect](setstatus-duplicate-sourceeffect.md) -- SetStatus duplicate-status check only matched ActiveMove, missing Synchronize's Condition sourceEffect and suppressing the `-fail` message.
- [shared-secondary-effect-mutation](shared-secondary-effect-mutation.md) -- Secondary effect objects were shared references to library templates, causing in-place mutations (e.g., Serene Grace) to leak across battles.
- [simultaneous-faint-win-condition](simultaneous-faint-win-condition.md) -- Simultaneous faint win condition was reversed, awarding the win to the wrong side.
- [singleevent-relayvar-default](singleevent-relayvar-default.md) -- `SingleEvent` did not default `relayVar` to `true` when called without one, causing null relay variables in downstream handlers.
- [sleep-clause-purifying-salt](sleep-clause-purifying-salt.md) -- Sleep Clause check ran before RunEvent(SetStatus), preventing Purifying Salt from blocking sleep first and producing incorrect protocol output.
- [slow-start-onend-handler](slow-start-onend-handler.md) -- Slow Start was missing its OnEnd handler, so the `|-end|Slow Start|[silent]` message was not emitted on faint.
- [spreadmovehit-double-secondaries](spreadmovehit-double-secondaries.md) -- SpreadMoveHit processed secondaries twice when called from the selfDrops path, consuming extra PRNG calls.
- [spurious-immune-after-protection](spurious-immune-after-protection.md) -- Protection moves emitted spurious `|-immune|` messages, and Heal Block (Psychic Noise) was missing its OnBeforeMove handler to block healing moves.
- [status-pattern-matching-null](status-pattern-matching-null.md) -- Status-immunity abilities matched `Status: not ConditionId.None` which incorrectly treated `null` as a primary status, showing immunity messages on secondary effects.
- [stomping-tantrum-protect-doubling](stomping-tantrum-protect-doubling.md) -- Stomping Tantrum incorrectly doubled after Protect because the protection path set `MoveThisTurnResult = false`.
- [substitute-blocking-status-tryprimaryhit](substitute-blocking-status-tryprimaryhit.md) -- Substitute did not block status moves (Yawn, Encore, etc.) because TryPrimaryHit's `null` return was not converted to `NullRelayVar`.
- [substitute-target-propagation](substitute-target-propagation.md) -- SpreadMoveHit's modified targets (nullified by Substitute) were discarded, causing incorrect `timesAttacked` increments and Rage Fist damage.
- [swapped-on-try-hit-params](swapped-on-try-hit-params.md) -- Soundproof, Sap Sipper, and Sturdy had swapped source/target parameters in their OnTryHit handlers.
- [switchin-onstart-fallback](switchin-onstart-fallback.md) -- OnStart-to-OnSwitchIn fallback used the wrong cache key, causing items like White Herb to fire twice at the wrong priority.
- [synchronize-duplicate-fail](synchronize-duplicate-fail.md) -- SetStatus duplicate-status check did not handle Synchronize's Condition sourceEffect, suppressing the `-fail` message for already-statused targets.
- [synchronize-immune-message](synchronize-immune-message.md) -- Synchronize did not trigger the `|-immune|` message when attempting to inflict status on a type-immune target.
- [synchronize-status-attribution](synchronize-status-attribution.md) -- Synchronize passed `null` as sourceEffect instead of the status Condition, causing incorrect attribution and missing immune messages.
- [throat-chop-residual-ordering](throat-chop-residual-ordering.md) -- Throat Chop's residual fired at the default order instead of the correct `order: 22`, misplacing its end message in the protocol log.
- [tryaddvolatile-null-to-falsy](tryaddvolatile-null-to-falsy.md) -- Prefixed TryAddVolatile handlers did not convert `null` returns to `NullRelayVar`, allowing volatiles to be added when handlers intended to block them.
- [tryhit-notfail-vs-null](tryhit-notfail-vs-null.md) -- TryHit wrappers mapped both NOT_FAIL and null to NullRelayVar, losing the distinction needed for Stomping Tantrum's failure detection after ability-absorbed moves.
- [tryhitside-null-falsy](tryhitside-null-falsy.md) -- TryHitSide/TryHitField did not treat null as falsy, causing Magic Bounce to apply both the bounced and original move's effects (e.g., double Sticky Web).
- [twoturnmove-volatile-persistence](twoturnmove-volatile-persistence.md) -- Two-turn moves removed the generic TwoTurnMove volatile in OnTryMove instead of the move-specific one, and 5 marker conditions were missing from the library.
- [unaware-modifyboost-return](unaware-modifyboost-return.md) -- Unaware's ModifyBoost handler returned VoidReturn instead of the modified boosts, discarding its accuracy/evasion zeroing.
- [uturn-selfswitch-disguise-zero](uturn-selfswitch-disguise-zero.md) -- U-turn selfSwitch did not trigger when Disguise reduced damage to 0, due to missing selfSwitch didSomething logic and a guarded didAnything update.
- [water-spout-zero-basepower](water-spout-zero-basepower.md) -- Water Spout/Eruption/Dragon Energy computed base power 0 at very low HP due to integer division truncation, instead of clamping to minimum 1.
- [weather-property-hiding-active-move](weather-property-hiding-active-move.md) -- ActiveMove's `Weather` property hid the inherited one, and `Reset()` did not copy the value from the template, so weather moves had no effect.
- [weather-reactivation-same-active](weather-reactivation-same-active.md) -- Setting weather that was already active sometimes re-activated it instead of silently returning false.
- [weighthg-fractional-truncation](weighthg-fractional-truncation.md) -- WeightHg cast to int before multiplying by 10, truncating fractional kilograms and producing incorrect weight values for Heavy Slam and Grass Knot.
- [wish-missing-slotcondition](wish-missing-slotcondition.md) -- Wish's move definition was missing its `SlotCondition` property, so it had no delayed healing effect.
